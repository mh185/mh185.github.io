# 电影文章写作与发布标准 (V1.0)

> **适用范围**：网站内所有单片深度影评、导演专题、佳片推荐列表、电影节专题文章。后续新增或改写电影类内容，必须严格按本标准执行。

---

## 一、 核心目标

高质量的电影文章不能流于简单的“剧情简介+主观情绪宣泄”，必须做到：

1. **有深度的文本拆解**：不仅写“讲了什么故事”，更要写“如何讲好故事”（叙事结构、视听语言、隐喻符号）。
2. **客观理性的审美态度**：避免极端、浮夸的赞美或贬低（如“世纪神作”、“垃圾至极”），多用镜头分析、影史定位与行业背景说话。
3. **沉浸式视听结合**：文章必须配置高清剧照，且**在核心段落嵌入高清晰度、无卡顿的官方预告片或经典镜头解析视频**，提升阅读体验。
4. **长期检索与收藏价值**：文章应具备SEO友好度，适合长期作为读者观影决策的参考指南。

---

## 二、 基础量化指标

- **字数底线**：单片深度影评/专题解析正文不少于 **1800字**；精选/重点推荐文章不少于 **2500字**。
- **媒体配置**：
  - **海报/首图**：1张标准比例电影海报（优先竖图/海报）。
  - **正文插图**：不少于 **4张** 高清电影剧照（禁止带播放器水印、低清截图）。
  - **外链视频**：必须嵌入至少 **1个** 官方高清预告片或经典场景分析视频（支持 Bilibili、YouTube 等主流平台 iframe 嵌入）。
- **标签规范**：必须包含：电影名、导演、年份、类型、影评/推荐。

---

## 三、 Frontmatter 标准格式

每篇电影文章的 Markdown 头部必须包含以下元数据：

```yaml
---
draft: false
title: 《电影名》+ 核心审美切入点（拒绝标题党）
description: 简明提炼本片的核心看点、视听特色，并说明文章包含深度文本解析与[预告片/名场面]视频。
pubDate: YYYY-MM-DD
license: cc-by-nc-sa-4-0
tags:
  - 电影名
  - 导演名
  - 电影类型（如：科幻/悬疑/文艺）
  - 国家/地区
  - 深度影评
image:
  src: /images/movies/slug/poster.webp
  alt: 《电影名》官方中文海报/经典剧照
video:
  platform: bilibili # 或 youtube/tencent 等
  embed_id: BVxxxxxxxxx # 外部平台视频唯一标识ID
---
```

## 四、 推荐文章结构

### 1. 导语（300字内）

- **要求**：直接切入电影的核心特质、所获重要荣誉（如奥斯卡、戛纳奖项）或当下的文化现象。
- **要点**：交代电影基本面（导演、主演、上映年份），用一句话阐明“这部电影为什么值得我们在今天重新讨论”。

### 2. 剧情主线与叙事张力

- **要求**：用克制的语言勾勒故事大纲。**严禁事无巨细地全片剧透**，重点写“冲突是如何建立的”。
- **避雷**：禁止直接搬运百度百科或维基百科的剧情大纲。

### 3. 核心主题与社会隐喻（深度解析）

- **要求**：拆解电影冰山之下的深层内核。
- **切入点**：
  - 角色的心理隐喻与成长弧光。
  - 电影所折射的时代背景或社会学议题。
  - 导演一贯的个人母题（如诺兰的“时间与记忆”、李安的“父子与东西方冲突”）。

### 4. 视听语言与导演美学（技术拆解）

- **要求**：必须有一段针对“电影感”的专业分析，拒绝门外汉式的空洞夸赞。
- **分析维度**：色彩美学（冷暖色调暗示）、镜头设计（长镜头、主观视角）、配乐与音效设计。

### 5. 经典名场面拆解与视频嵌入（互动板块）

- **要求**：在此板块必须嵌入外链视频。文字需详细解析该视频片段中的演员表演细节或镜头剪辑逻辑，形成“**边看视频，边读解析**”的沉浸式体验。

## 五、 外链视频嵌入技术标准

为了保证网站加载性能、响应式适配（移动端不缩水）以及跨平台兼容性，外链视频必须遵循以下嵌入规范：

### 1. 视频平台选择优先级

1.  **Bilibili (哔哩哔哩)**：国内用户首选，加载速度快，弹幕可控，资源丰富。
2.  **YouTube**：面向海外用户或国际化版本网站的首选。
3.  **腾讯视频/爱奇艺**：作为备选官方预告片源。

### 2. 响应式 HTML 嵌入代码模板

不要直接复制第三方平台生成的绝对宽度代码。为了实现手机端和电脑端的完美自适应，必须使用以下 **16:9 比例的响应式容器** 包裹 iframe：

```html
<div
  class="video-wrapper"
  style="position: relative; padding-bottom: 56.25%; height: 0; overflow: hidden; max-width: 100%; background: #000; border-radius: 8px; margin: 20px 0;"
>
  <!-- 以 Bilibili 嵌入为例 -->
  <iframe
    src="//player.bilibili.com/player.html?bvid=BV1xx411c7m9&page=1&high_quality=1&as_wide=1"
    scrolling="no"
    border="0"
    frameborder="no"
    framespacing="0"
    allowfullscreen="true"
    loading="lazy"
    style="position: absolute; top: 0; left: 0; width: 100%; height: 100%; border: 0;"
  >
  </iframe>
</div>
<p style="text-align: center; font-size: 13px; color: #666; margin-top: -10px;">
  <i>视频：官方高清预告片（如无法播放，请点击跳转原平台观看）</i>
</p>
```

#### 关键参数配置说明：

- `high_quality=1`：强制请求高清画质。
- `as_wide=1`：强制宽屏播放。
- `loading="lazy"`：**必须配置**，启用浏览器原生懒加载，不影响文章首屏文字和图片的加载速度。
- `allowfullscreen="true"`：允许用户全屏播放。

## 六、 内容质量红线（正反案例对比）

### 1. 文字态度

- ❌ **禁止空泛赞美**：“特效美哭了”、“演技炸裂”、“神作不解释”。
- **推荐专业词汇**：“低饱和度的冷调冷光奠定了本片的悬疑底色”、“面部微表情的长镜头特写完成了人物内心的挣扎转变”、“非线性叙事结构的精妙拼贴”。

### 2. 视频关联度

- ❌ **禁止无意义堆砌**：插入与文章解析不匹配的超长全片录像、画质低劣的盗版资源或带有大量个人推广片头片尾的剪辑视频。
- **推荐视频源**：电影制片方官方发布的「Theatrical Trailer (院线预告片)」、官方授权的「Inside Look (幕后特辑)」或专业影评人无版权争议的「Spotlight Scene Analysis (单镜头拆解)」。

---

## 七、 质量自检清单

发布前，请编辑务必逐项勾选确认：

- [ ] 正文字数已达到标准（普通 >=1800字，精选 >=2500字）。
- [ ] 电影年份、导演、编剧、主演等核心基础信息核对无误。
- [ ] YAML 元数据完整，且 `image.src` 海报图存在并已转换为 WebP 格式。
- [ ] 至少包含 4 张与当前文字段落高度相关的剧照插图。
- [ ] 至少嵌入了 1 个外链视频，且视频已包裹在 `video-wrapper` 响应式容器内。
- [ ] 实测视频在 **手机移动端** 和 **电脑端** 均能正常自适应，且播放无明显延迟。
- [ ] 影评观点具有独特性，无明显抄袭、洗稿豆瓣或知乎的热门影评。
