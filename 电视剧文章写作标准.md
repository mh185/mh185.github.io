# 电视剧文章写作与发布标准 (V1.0)

> **适用范围**：网站内所有单剧深度剧评、剧集推荐列表、主创/演员专题、平台片单专题文章（美剧、英剧、日剧、韩剧、国产剧、动画剧集等）。后续新增或改写电视剧类内容，必须严格按本标准执行。
>
> 本标准与《电影文章写作标准》配套。电视剧与电影最大的不同在于：**篇幅长（多季多集）、叙事线长（人物弧光跨季展开）、有"完结/连载"状态、观看是"追"而不是"看"**。因此本标准在电影标准的基础上，额外强化"分季评价、入坑与弃剧指南、追剧节奏、剧透管理"四大板块。

---

## 一、 核心目标

高质量的剧评不能停留在"剧情复述 + 主观安利"，必须做到：

1. **有深度的文本拆解**：不仅写"讲了什么"，更要写"这部剧如何用几十小时的体量把故事讲好"（长线叙事结构、人物群像的塑造、季与季之间的节奏控制）。
2. **客观理性的审美态度**：避免"封神/烂尾"式的极端情绪化结论；要用具体的剧集、具体的人物转变、具体的制作水准说话。评价"烂尾"也要讲清楚烂在哪、为什么。
3. **对"追剧决策"负责**：读者看剧评，核心诉求是"这剧值不值得我投入几十个小时"。文章必须明确交代**完结状态、季数集数、入坑门槛、弃剧点、观看节奏和观看平台**，帮读者做决定。
4. **沉浸式视听结合**：必须配高清剧照，并在核心段落嵌入**官方高清预告片（Trailer）、片头（Opening）或名场面片段**。
5. **长期检索与收藏价值**：SEO 友好，适合长期作为读者的"追剧指南"被搜索到。

---

## 二、 基础量化指标

- **字数底线**：单剧深度剧评/专题解析正文不少于 **1800字**；精选/重点推荐（如"镇站之宝"级神剧）不少于 **2500字**。
- **媒体配置**：
  - **海报/首图**：1张标准比例剧集海报（优先竖版官方海报或 Key Art）。
  - **正文插图**：不少于 **4张** 高清剧照（禁止带播放器水印、弹幕、低清截图；优先能体现人物关系与质感的官方剧照）。
  - **外链视频**：必须嵌入至少 **1个** 官方高清预告片、正式片头或名场面片段（支持 Bilibili、YouTube 等主流平台 iframe 嵌入）。
- **标签规范**：必须包含：**剧名、主创/导演、首播年份、类型、国别/地区、首播平台、追剧、剧评/电视剧推荐**。（"电视剧推荐"标签用于 SEO 聚合，务必带上。）

## 三、 Frontmatter 标准格式

每篇电视剧文章的 Markdown 头部必须包含以下元数据：

```yaml
---
draft: false
title: 《剧名》+ 核心审美切入点（拒绝标题党）
description: 简明提炼本剧的核心看点、类型定位、完结状态与追剧价值，并说明文章包含深度解析与[预告片/名场面]视频。60-90字，利于SEO。
pubDate: YYYY-MM-DD
# rank: N            # 若属于某个排行榜（如"必刷高分剧Top100"），填入名次，用于列表排序与"精选第N名"徽章
license: cc-by-nc-sa-4-0
tags:
  - 剧名
  - 主创/导演名（showrunner 或核心导演）
  - '首播年份'          # 如 '2008'
  - 剧集类型（如：犯罪/科幻/悬疑/历史/生活）
  - 国别（如：美国电视剧/英国电视剧/日本电视剧/韩国电视剧/国产剧）
  - 首播平台（如：HBO/Netflix/Apple TV+ 等，可选）
  - 追剧
  - 剧评
  - 电视剧推荐
image:
  src: /images/tv/slug/poster.webp
  alt: 《剧名》官方海报/经典剧照
video:
  platform: bilibili   # 或 youtube/tencent 等
  embed_id: BVxxxxxxxxx # 外部平台视频唯一标识ID
---
```

> **说明**：`slug` 用剧集英文名的短横线小写形式（如 `breaking-bad`、`game-of-thrones`）；图片统一放在 `public/images/tv/<slug>/` 下，海报 `poster.webp`、剧照 `scene-1.webp … scene-N.webp`，并生成 `_thumbs` 缩略图供卡片使用。

## 四、 推荐文章结构

### 1. 导语（300字内）

- **要求**：直接切入本剧的核心特质、重要荣誉（艾美奖、金球奖、豆瓣/IMDb 高分）或文化现象。
- **必须交代四件事**：①主创/主演；②首播年份与**首播平台**；③**季数与完结状态**（已完结 X 季 / 连载中，第 X 季更新中 / 已砍）；④一句话说清"这部剧为什么值得投入时间"。

### 2. 设定与开篇钩子（不剧透）

- **要求**：用克制的语言交代世界观、类型定位和"第一集/第一季"如何建立钩子。
- **重点**：写清楚"它靠什么把观众留下来"，而不是逐集复述。
- **避雷**：**严禁跨季全剧透**；涉及关键反转、角色生死、结局走向时，一律写"**此处不剧透**"。

### 3. 分季评价与叙事节奏（剧集专属·核心板块）

- **要求**：这是剧评区别于影评的关键。要对**各季质量的起伏**给出判断：哪一季是巅峰、哪一季掉线、节奏是渐入佳境还是虎头蛇尾。
- **要点**：
  - 长线叙事是如何铺陈和收束的（有没有"填坑"，有没有"注水"）。
  - 若存在争议结局或"烂尾"讨论，要**客观说明争议点在哪**，给出自己的判断，但**不剧透具体情节**。
  - 连载剧要说明"目前更新到哪、后续值不值得等"。

### 4. 人物群像与角色弧光（深度解析）

- **要求**：电视剧的灵魂是"人随时间改变"。必须拆解至少 1-2 个核心角色跨季的成长/堕落弧光，以及群像关系的张力。
- **切入点**：主角的转变逻辑、反派的复杂性、配角的高光、演员的长线表演如何撑起角色。

### 5. 核心主题与社会议题（深度解析）

- **要求**：拆解剧集冰山之下的内核——它借这个故事在谈什么（权力、道德、阶级、科技、家庭、时代创伤等）。
- **切入点**：主创一贯的母题、剧集折射的社会背景、它为何能引发广泛共鸣。

### 6. 制作水准与视听语言（技术拆解）

- **要求**：必须有一段针对"剧集工业化水准"的专业分析，拒绝空洞夸赞。
- **分析维度**：摄影与色调、配乐与主题曲、片头设计（Title Sequence）、美术与场景、单集导演调度、电影级制作与电视叙事的结合。

### 7. 经典名场面拆解与视频嵌入（互动板块）

- **要求**：在此板块必须嵌入外链视频（官方预告片 / 名场面 / 片头）。文字需详细解析该片段中的表演细节、剪辑逻辑或情绪调度，形成"**边看视频，边读解析**"的沉浸式体验。**注意所选片段本身不得剧透关键反转。**

### 8. 入坑与追剧指南（剧集专属·实用板块）

- **要求**：给出可执行的"追剧决策建议"，这是读者最需要的部分。
- **必答清单**：
  - **入坑门槛**：前几集是否劝退？大概到第几集/第几季"封神"？值不值得熬过前期。
  - **弃剧点提示**：如果某季明显掉线，诚实提示（如"追到第X季即可，后续可选看"）。
  - **观看节奏**：适合一口气刷（binge）还是慢慢追？是"下饭剧"还是"需要专注"？
  - **观看平台**：国内外正版观看渠道（如有靠谱的官方/授权渠道就写）。
  - **完结提醒**：是否已完结、要不要等填坑，避免读者入了连载大坑。

### 9. 适合谁看 / 为什么值得追（收尾）

- 分 2-3 类人群说明适配度，最后升华：这部剧在同类剧集或剧史中的位置与价值。

## 五、 外链视频嵌入技术标准

为保证加载性能、响应式适配（移动端不缩水）及跨平台兼容，外链视频必须遵循以下规范：

### 1. 视频平台选择优先级

1. **Bilibili（哔哩哔哩）**：国内用户首选，加载快、资源丰富。
2. **YouTube**：面向海外/国际化版本首选（官方频道的 Official Trailer / Opening Credits）。
3. **腾讯视频 / 爱奇艺 / 优酷**：作为备选官方预告片源。

> **可嵌入性校验**：YouTube 视频用 oembed 接口校验（`https://www.youtube.com/oembed?url=...&format=json` 返回 200 即可嵌入）；确认所选为**官方/发行方**上传，非盗版或带个人推广片头的剪辑。

### 2. 响应式 HTML 嵌入代码模板（16:9 自适应）

不要直接复制第三方生成的绝对宽度代码，必须用 **16:9 响应式容器** 包裹 iframe：

```html
<div
  class="video-wrapper"
  style="position: relative; padding-bottom: 56.25%; height: 0; overflow: hidden; max-width: 100%; background: #000; border-radius: 8px; margin: 20px 0;"
>
  <!-- YouTube 示例 -->
  <iframe
    src="https://www.youtube.com/embed/VIDEOID"
    title="《剧名》官方预告片"
    loading="lazy"
    allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share"
    allowfullscreen
    style="position: absolute; top: 0; left: 0; width: 100%; height: 100%; border: 0;"
  ></iframe>
</div>
<p style="text-align: center; font-size: 13px; color: #666; margin-top: -10px;">
  <i>视频：《剧名》官方预告片（如无法播放，请点击跳转原平台观看）</i>
</p>
```

```html
<!-- Bilibili 备用示例：把 src 换成 //player.bilibili.com/player.html?bvid=BVxxxx&high_quality=1&as_wide=1 -->
```

#### 关键参数说明：

- `loading="lazy"`：**必须配置**，启用原生懒加载，不拖慢首屏。
- `allowfullscreen`：允许全屏。
- Bilibili 参数 `high_quality=1`（强制高清）、`as_wide=1`（强制宽屏）。

## 六、 内容质量红线（正反案例对比）

### 1. 文字态度

- ❌ **禁止空泛结论**："封神！"、"史上最强美剧"、"烂尾到不想说话"、"演技炸裂"。
- ✅ **推荐专业表达**："第三季通过多线并进把权力斗争推向高潮"、"主角从化学老师到毒枭的转变，靠的是每一季一次道德底线的松动"、"冷蓝色调与手持摄影强化了纪实感与压迫感"、"结局的争议主要来自节奏仓促而非逻辑硬伤"。

### 2. 剧透纪律（剧集红线，比电影更严）

- ❌ **禁止跨季剧透**：不得透露关键人物生死、重大反转、结局走向。
- ✅ 遇到关键节点统一用"**此处不剧透**"带过；名场面视频也要选**不含致命剧透**的片段。

### 3. 视频关联度

- ❌ **禁止无意义堆砌**：整集录像、盗版低清资源、带大量个人推广片头片尾的剪辑、含结局剧透的"名场面合集"。
- ✅ **推荐视频源**：官方 Season Trailer（分季预告）、官方 Title Sequence（片头）、官方授权的 Inside the Episode（幕后），或无版权争议的单场景解析。

### 4. 原创性

- ❌ 禁止洗稿豆瓣、知乎、B站热门长评。
- ✅ 观点、分析、遣词造句必须原创；不复述剧情大纲、不抄台词旁白原文。

## 七、 质量自检清单

发布前，请编辑务必逐项勾选确认：

- [ ] 正文字数达标（普通 ≥1800字，精选 ≥2500字）。
- [ ] 导语已交代主创、首播年份、**首播平台**、**季数与完结状态**。
- [ ] 有独立的**分季评价/叙事节奏**分析（不是逐集流水账）。
- [ ] 有至少 1-2 个核心角色的**跨季弧光**解析。
- [ ] 有专门的**入坑与追剧指南**（入坑门槛 / 弃剧点 / 观看节奏 / 观看平台 / 完结提醒）。
- [ ] 剧集核心信息（季数、集数、平台、主创、主演、完结状态）核对无误。
- [ ] YAML 元数据完整，`image.src` 海报存在且为 WebP，已生成 `_thumbs` 缩略图。
- [ ] 至少 4 张与当前段落高度相关的高清剧照，无水印/弹幕。
- [ ] 至少嵌入 1 个官方外链视频，包裹在 `video-wrapper` 响应式容器内，手机端和电脑端均自适应、无明显延迟。
- [ ] **全程无关键剧透**，敏感节点已用"此处不剧透"处理。
- [ ] 观点原创，无抄袭/洗稿。
